# Theme Design: validation\n\n## Goal\nAdd input validation utilities and custom error types. This increases codebase complexity and tests auto-dev's ability to modify existing code (not just add new files).\n\n## Context\nv001 established project structure, v002 added utility functions. v003 refactors those utilities to use proper error handling.\n\n## Architecture Decisions\n- **Error hierarchy**: Base ValidationError with specific subclasses\n- **Type guards**: Functions that narrow types for TypeScript\n- **Assertions**: Functions that throw on invalid input\n- **Backward compatibility**: Existing valid inputs still work\n\n## Feature Sequence\n1. `001-error-types` - Create error class hierarchy (no dependencies)\n2. `002-validators` - Create validation functions (uses errors)\n3. `003-integrate-validation` - Refactor existing utils (uses errors, modifies existing files)\n\n## Success Criteria\n- Custom error types exported from src/index.ts\n- Validation utilities exported from src/index.ts\n- Existing utils throw typed errors\n- All tests pass\n- Build succeeds
